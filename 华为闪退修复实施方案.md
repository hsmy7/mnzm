## 一、华为 AGC Crash SDK 接入（预计 3 小时）

**背景**：
当前自研 `CrashHandler`（`core/CrashHandler.kt`）仅能捕获 Java 层异常并写入本地文件。Release 包经过 R8 混淆后，堆栈不可读，**无法定位华为设备上的真实崩溃原因**。华为 AGC Crash 服务可自动上传符号表并还原混淆堆栈。

**行业对标**：

| 产品 | 崩溃 SDK | 符号化 | 华为设备适配 |
|------|----------|--------|-------------|
| 原神 | Bugly + 自研 | ✅ 自动上传 mapping | HarmonyOS 专项适配 |
| 王者荣耀 | Bugly 专业版 | ✅ | EMUI 深度适配 |
| 崩坏星铁 | Firebase Crashlytics + 华为 AGC Crash | ✅ 双通道 | HMS Core 内置 |
| 本游戏（当前） | 自研 CrashHandler | ❌ 无符号化 | 无华为适配 |

**参照标准**：米哈游系产品（原神/星铁）采用双通道崩溃收集 — Firebase Crashlytics（海外）+ 华为 AGC Crash（国内华为渠道），确保全设备覆盖。

#### 步骤 3：在 Application 中初始化 AGC Crash

**文件**：`android/app/src/main/java/com/xianxia/sect/XianxiaApplication.kt`

在 `onCreate()` 开头、`super.onCreate()` 之后添加：

```kotlin
import com.huawei.agconnect.crash.AGConnectCrash

override fun onCreate() {
    super.onCreate()
    instance = this

    // 华为 AGC Crash：自动采集崩溃日志并上传符号表
    // 仅在华为/荣耀设备上启用，其他设备继续使用自研 CrashHandler
    if (DeviceCompatibilityHelper.isHuaweiOrHonor) {
        try {
            AGConnectCrash.getInstance().enableCrashCollection(true)
            AGConnectCrash.getInstance().setUserId(sessionManager?.userId ?: "unknown")
            Log.i(TAG, "AGC Crash enabled for Huawei device")
        } catch (e: Exception) {
            Log.w(TAG, "AGC Crash init failed, falling back to self-built handler", e)
        }
    }

    // ... 后续原有初始化逻辑 ...
}
```

#### 步骤 4：自定义崩溃上报（补充上下文）

在 `CrashHandler` 中增加一个接口，当自研崩溃处理器捕获到异常时，同步上报到 AGC：

**文件**：`android/app/src/main/java/com/xianxia/sect/core/CrashHandler.kt`

在 `uncaughtException()` 方法的 `performEmergencySave()` 之前添加：

```kotlin
override fun uncaughtException(thread: Thread, throwable: Throwable) {
    Log.e(TAG, "Uncaught exception in thread ${thread.name}", throwable)

    try {
        // 上报到华为 AGC Crash（如果可用）
        if (DeviceCompatibilityHelper.isHuaweiOrHonor) {
            try {
                AGConnectCrash.getInstance().recordException(throwable)
            } catch (_: Exception) { /* 静默失败，不影响自研流程 */ }
        }

        // 1. 尝试紧急保存
        performEmergencySave()
        // ... 后续原有逻辑 ...
    }
}
```

#### 步骤 5：上传符号表实现堆栈还原

在 `android/app/build.gradle` 中添加符号表自动上传配置：

```groovy

// 华为 AGC 符号表自动上传
```

## 三、ProGuard/R8 规则完善（预计 2 小时）

**背景**：
当前 `proguard-rules.pro` 中缺少对 Compose 运行时反射、Hilt 生成代码、以及 `@Serializable` data class 的显式 `@Keep` 保护。华为 HarmonyOS 的 AOT 编译比标准 ART 更激进，更容易触发反射失败。

### 实施步骤

#### 步骤 1：为所有 `@Serializable` 数据类添加 `@Keep`

遍历所有包含 `@Serializable` 注解的 data class，添加 `@Keep`：

```kotlin
import androidx.annotation.Keep

@Keep
@Serializable
data class Disciple(...)
```

```bash
# 查找所有需要添加 @Keep 的文件
grep -rn "@Serializable" android/app/src/main/java/com/xianxia/sect/core/model/ \
  android/app/src/main/java/com/xianxia/sect/data/model/
```

**涉及文件清单**：
- `core/model/Disciple.kt`
- `core/model/DiscipleAggregate.kt`
- `core/model/GameData.kt`
- `data/model/SaveSlot.kt`
- 以及所有 `@Serializable` 的嵌套数据类

#### 步骤 2：补充 ProGuard keep 规则

在 `proguard-rules.pro` 末尾追加：

```proguard
# === 华为 HarmonyOS AOT 兼容规则 ===

# Compose runtime stability annotations
-keep @interface androidx.compose.runtime.Stable
-keep @interface androidx.compose.runtime.Immutable
-keepclasseswithmembers @androidx.compose.runtime.Stable class * { *; }
-keepclasseswithmembers @androidx.compose.runtime.Immutable class * { *; }

# Hilt generated components
-keep class * extends dagger.hilt.android.internal.managers.ViewComponentManager$FragmentContextWrapper { *; }
-keep class * extends dagger.hilt.android.internal.managers.ActivityComponentManager$ActivityContextWrapper { *; }

# Manual proficiency — accessed via reflection
-keep class com.xianxia.sect.core.model.ManualProficiencyData { *; }

# Room TypeConverters
-keep class * extends androidx.room.TypeConverter { *; }

# Compose runtime internal reflection
-keep class kotlin.coroutines.Continuation { *; }
-keepclassmembers class kotlinx.coroutines.** {
    volatile <fields>;
}

# ReLinker native methods
-keep class com.getkeepsafe.relinker.** { *; }
```

## 四、混淆诊断能力（预计 1 小时）

**背景**：
即使接入 AGC Crash，混淆堆栈的可读性仍依赖 mapping 文件。当前无机制将版本号与 mapping 关联。

### 实施步骤

#### 步骤 1：保留 mapping 文件并随版本发布

在 `android/app/build.gradle` 中：

```groovy
// 构建后将 mapping 文件复制到固定目录
tasks.whenTaskAdded { task ->
    if (task.name == 'minifyReleaseWithR8') {
        task.doLast {
            def mappingFile = file("${buildDir}/outputs/mapping/release/mapping.txt")
            if (mappingFile.exists()) {
                def destDir = file("${rootDir}/mappings/${android.defaultConfig.versionName}")
                destDir.mkdirs()
                copy {
                    from mappingFile
                    into destDir
                    rename { "mapping-${android.defaultConfig.versionName}.txt" }
                }
            }
        }
    }
}
```

#### 步骤 2：远程日志上传

在 `CrashHandler` 中增加将崩溃日志上传到自有服务器的能力：

```kotlin
// 在 CrashHandler.kt 中新增
private fun tryUploadCrashLog(crashLogFile: File?) {
    try {
        if (crashLogFile == null || !crashLogFile.exists()) return
        val content = crashLogFile.readText().take(8000)

        Thread {
            try {
                val client = OkHttpClient.Builder()
                    .connectTimeout(3, TimeUnit.SECONDS)
                    .build()
                val body = okhttp3.FormBody.Builder()
                    .add("version", BuildConfig.VERSION_NAME)
                    .add("device", "${Build.MANUFACTURER} ${Build.MODEL}")
                    .add("sdk", Build.VERSION.SDK_INT.toString())
                    .add("stack", content)
                    .build()
                val request = okhttp3.Request.Builder()
                    .url("${BuildConfig.API_BASE_URL}crash-report")
                    .post(body)
                    .build()
                client.newCall(request).execute().close()
            } catch (_: Exception) { /* 静默失败 */ }
        }.start()
    } catch (_: Exception) { /* 静默失败 */ }
}
```

### 技术路线

| 方案 | 工作量 | 维护成本 | 推荐场景 |
|------|--------|----------|----------|
| **A. 纯 ArkTS 重写** | 6-12 人月 | 高（双代码库） | 华为 DAU > 30% |
