# 游戏闪退与存档损坏问题修复 Spec

## Why

用户反馈正常游玩时游戏会随机闪退，并且存档文件损坏无法读取。这是一个严重的数据安全问题，需要：

1. 增强异常处理，防止游戏崩溃
2. 确保存档写入的原子性，防止存档损坏
3. 添加存档损坏检测和恢复机制
4. 增强日志记录，便于问题排查

## What Changes

* 增强游戏循环异常处理，防止单个tick异常导致崩溃

* 确保存档写入使用原子操作（临时文件+重命名）

* 添加存档损坏检测和自动恢复机制

* 增强全局异常捕获，崩溃前尝试保存数据

* 添加详细的日志记录

## Impact

* Affected specs: 存档系统、游戏生命周期管理、游戏循环

* Affected code:

  * `GameActivity.kt` - 异常处理和崩溃恢复

  * `GameViewModel.kt` - 游戏循环异常处理

  * `GameEngine.kt` - 核心逻辑异常处理

  * `SaveManager.kt` - 存档原子性和损坏恢复

## ADDED Requirements

### Requirement: 游戏循环异常容错

游戏循环中的异常不应导致游戏崩溃，应记录日志并继续运行。

#### Scenario: 游戏循环异常处理

* **WHEN** 游戏循环中某个tick处理发生异常

* **THEN** 系统应捕获异常、记录详细日志

* **AND** 游戏应继续运行而不是崩溃

* **AND** 异常信息应包含游戏状态便于排查

### Requirement: 存档写入原子性

存档操作必须是原子的，防止写入过程中崩溃导致存档损坏。

#### Scenario: 存档写入中途崩溃保护

* **WHEN** 存档写入过程中发生崩溃

* **THEN** 系统应保留之前的完整存档

* **AND** 不应产生损坏的存档文件

* **AND** 下次启动时应能正常加载之前的存档

#### Scenario: 存档写入成功

* **WHEN** 存档写入完成

* **THEN** 新存档应原子性地替换旧存档

* **AND** 备份文件应保留以备恢复

### Requirement: 存档损坏检测与恢复

系统应能检测存档损坏并尝试自动恢复。

#### Scenario: 检测损坏存档

* **WHEN** 用户尝试加载存档

* **AND** 存档文件损坏（校验失败或解析失败）

* **THEN** 系统应提示用户存档损坏

* **AND** 提供从备份恢复的选项

#### Scenario: 自动恢复备份

* **WHEN** 存档损坏且存在有效备份

* **THEN** 系统应自动尝试从备份恢复

* **AND** 恢复成功后提示用户

### Requirement: 全局异常捕获与数据保护

系统应在发生未捕获异常时，尽可能保存当前游戏数据。

#### Scenario: 游戏崩溃时保护数据

* **WHEN** 游戏发生未捕获的异常导致崩溃

* **THEN** 系统应尝试在崩溃前保存当前游戏数据到紧急存档

* **AND** 记录崩溃日志便于排查

### Requirement: 详细日志记录

关键游戏操作应记录详细日志，便于问题排查。

#### Scenario: 关键操作日志

* **WHEN** 执行关键游戏操作（存档、读档、游戏循环等）

* **THEN** 系统应记录详细日志

* **AND** 日志应包含操作结果和任何异常信息

## MODIFIED Requirements

### Requirement: SaveManager 存档写入

SaveManager 的存档写入必须保证原子性。

**原实现**: 使用临时文件，但可能存在并发问题
**新实现**:

* 使用写入锁确保无并发写入

* 临时文件写入完成后再原子重命名

* 保留备份文件用于恢复

* 添加校验和验证

### Requirement: 游戏循环异常处理

游戏循环应捕获并处理所有异常。

**原实现**: 部分异常被捕获
**新实现**: 所有tick操作都包装在try-catch中，记录详细日志

## Root Cause Analysis

根据用户反馈（正常游玩随机崩溃，存档损坏），可能的问题原因：

1. **游戏循环异常**: `processSecondTick()` 或 `advanceDay()` 中可能存在未处理的异常导致崩溃
2. **存档写入竞态**: 存档操作与游戏循环可能存在并发问题，导致写入不完整
3. **内存问题**: 大数据量时可能发生OOM导致崩溃
4. **特定游戏状态**: 某些特定游戏状态可能触发边界条件异常
5. **存档写入中断**: 崩溃时正在写入存档，导致文件损坏

## Investigation Points

1. 检查 `processSecondTick()` 和 `advanceDay()` 的所有代码路径
2. 检查存档操作的并发安全性
3. 检查是否有空指针、数组越界等常见异常
4. 检查内存使用情况
5. 添加更详细的日志记录
