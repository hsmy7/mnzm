# 送礼好感度百分比增长机制规格

## Why

当前送礼系统使用固定好感度增加（薄礼+6、厚礼+18、重礼+30、大礼+40），导致高好感度宗门送礼效果不明显，低好感度宗门送礼效果过于显著。需要改为按当前好感度的百分比增长，使送礼效果更加合理且与宗门等级相关。

## What Changes

- **BREAKING** 修改灵石送礼好感度计算公式：从固定值改为当前好感度的百分比增长
- 新增宗门等级与档位对应的好感度百分比配置
- 大型宗门和顶级宗门不支持薄礼和厚礼（已被拒绝）

## Impact

- Affected specs: sect-gift-favor-system
- Affected code: GiftConfig.kt, GameEngine.kt

## ADDED Requirements

### Requirement: 好感度百分比增长配置

系统应根据宗门等级和送礼档位计算好感度增长百分比。

#### Scenario: 小型宗门好感度增长
- **GIVEN** 小型宗门（level=0）
- **WHEN** 送薄礼
- **THEN** 好感度增加当前好感度的60%
- **WHEN** 送厚礼
- **THEN** 好感度增加当前好感度的100%
- **WHEN** 送重礼
- **THEN** 好感度增加当前好感度的200%
- **WHEN** 送大礼
- **THEN** 好感度增加当前好感度的300%

#### Scenario: 中型宗门好感度增长
- **GIVEN** 中型宗门（level=1）
- **WHEN** 送薄礼
- **THEN** 好感度增加当前好感度的50%
- **WHEN** 送厚礼
- **THEN** 好感度增加当前好感度的80%
- **WHEN** 送重礼
- **THEN** 好感度增加当前好感度的160%
- **WHEN** 送大礼
- **THEN** 好感度增加当前好感度的250%

#### Scenario: 大型宗门好感度增长
- **GIVEN** 大型宗门（level=2）
- **WHEN** 送重礼
- **THEN** 好感度增加当前好感度的100%
- **WHEN** 送大礼
- **THEN** 好感度增加当前好感度的160%
- **WHEN** 送薄礼或厚礼
- **THEN** 被拒绝（现有拒绝机制已覆盖）

#### Scenario: 顶级宗门好感度增长
- **GIVEN** 顶级宗门（level=3）
- **WHEN** 送重礼
- **THEN** 好感度增加当前好感度的80%
- **WHEN** 送大礼
- **THEN** 好感度增加当前好感度的50%
- **WHEN** 送薄礼或厚礼
- **THEN** 被拒绝（现有拒绝机制已覆盖）

### Requirement: 好感度计算公式

系统应使用百分比公式计算最终好感度。

#### Scenario: 好感度计算
- **GIVEN** 宗门当前好感度为C，送礼档位百分比为P%
- **WHEN** 送礼成功
- **THEN** 新好感度 = min(C + C * P / 100, 100)

#### Scenario: 好感度下限保护
- **GIVEN** 宗门当前好感度为0
- **WHEN** 送礼成功
- **THEN** 好感度至少增加1点（避免0 * 百分比 = 0的情况）

## MODIFIED Requirements

### Requirement: 灵石送礼配置修改

原有灵石送礼配置使用固定好感度值，现修改为百分比配置。

#### Scenario: 配置结构变更
- **GIVEN** SpiritStoneGiftTier数据类
- **WHEN** 修改配置
- **THEN** 移除固定favor字段，改为按宗门等级配置的百分比映射

## REMOVED Requirements

### Requirement: 固定好感度值配置

**Reason**: 改为百分比增长机制，不再使用固定值
**Migration**: 将GiftConfig中的favor字段改为百分比配置

## 核心数值表

### 好感度增长百分比（按宗门等级和档位）

| 宗门等级 | 薄礼 | 厚礼 | 重礼 | 大礼 |
|---------|------|------|------|------|
| 小型宗门(0) | 60% | 100% | 200% | 300% |
| 中型宗门(1) | 50% | 80% | 160% | 250% |
| 大型宗门(2) | - | - | 100% | 160% |
| 顶级宗门(3) | - | - | 80% | 50% |

注： "-" 表示该档位会被该等级宗门拒绝

### 好感度计算示例

假设当前好感度为20：

| 宗门等级 | 薄礼后 | 厚礼后 | 重礼后 | 大礼后 |
|---------|--------|--------|--------|--------|
| 小型宗门 | 32(+12) | 40(+20) | 60(+40) | 80(+60) |
| 中型宗门 | 30(+10) | 36(+16) | 52(+32) | 70(+50) |
| 大型宗门 | - | - | 40(+20) | 52(+32) |
| 顶级宗门 | - | - | 36(+16) | 30(+10) |

### 配置数据结构

```kotlin
// 好感度百分比配置：key为宗门等级，value为档位到百分比的映射
val FAVOR_PERCENTAGE_CONFIG = mapOf(
    // 小型宗门
    0 to mapOf(
        1 to 60,   // 薄礼
        2 to 100,  // 厚礼
        3 to 200,  // 重礼
        4 to 300   // 大礼
    ),
    // 中型宗门
    1 to mapOf(
        1 to 50,   // 薄礼
        2 to 80,   // 厚礼
        3 to 160,  // 重礼
        4 to 250   // 大礼
    ),
    // 大型宗门
    2 to mapOf(
        3 to 100,  // 重礼
        4 to 160   // 大礼
    ),
    // 顶级宗门
    3 to mapOf(
        3 to 80,   // 重礼
        4 to 50    // 大礼
    )
)
```
