# 修复储物袋界面道具卡片样式与自动学习功法问题 Spec

## Why
储物袋界面的道具卡片使用列表样式显示，与仓库界面的网格卡片样式不一致，影响用户体验的一致性。同时，弟子不会自动学习储物袋内的功法，导致玩家需要手动操作，降低了游戏的自动化体验。

## What Changes
- 统一储物袋界面的道具卡片样式，使其与仓库界面保持一致的网格卡片样式
- 修复弟子自动学习储物袋内功法的逻辑问题

## Impact
- Affected specs: 储物袋界面、弟子功法学习系统
- Affected code: 
  - `DiscipleDetailScreen.kt` - StorageBagDialog 组件
  - `GameEngine.kt` - processAutoLearnManual 函数

## ADDED Requirements

### Requirement: 储物袋界面道具卡片样式统一
系统应将储物袋界面的道具卡片样式改为与仓库界面一致的网格卡片样式。

#### Scenario: 储物袋界面显示道具卡片
- **WHEN** 用户打开弟子的储物袋界面
- **THEN** 道具应以网格卡片形式显示（4列布局）
- **AND** 卡片样式应与仓库界面一致（正方形卡片、稀有度边框颜色、选中高亮、查看按钮）
- **AND** 数量显示在卡片右下角

### Requirement: 弟子自动学习储物袋内功法
系统应确保弟子能够自动学习储物袋内的功法。

#### Scenario: 弟子储物袋内有未学习功法
- **GIVEN** 弟子储物袋内有功法道具
- **AND** 弟子功法槽位未满
- **WHEN** 每月处理时执行自动装备逻辑
- **THEN** 弟子应自动学习储物袋内最高品质的功法
- **AND** 功法从储物袋中移除

#### Scenario: 弟子功法槽位已满但储物袋有更高品质功法
- **GIVEN** 弟子功法槽位已满
- **AND** 储物袋内有比已学习功法更高品质的功法
- **WHEN** 每月处理时执行自动装备逻辑
- **THEN** 弟子应自动替换最低品质的功法为储物袋内最高品质的功法
- **AND** 被替换的功法返回储物袋

## MODIFIED Requirements

### Requirement: 储物袋道具卡片组件
将现有的列表样式道具卡片改为网格卡片样式，复用仓库界面的卡片设计模式。

## REMOVED Requirements
无
