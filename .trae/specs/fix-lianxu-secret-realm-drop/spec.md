# 修复炼虚境弟子秘境掉落品阶错误问题

## Why
用户反馈炼虚境弟子去秘境只掉落凡品道具，这是因为 `getRarityRangeByRealm` 方法中的条件判断边界值设置错误，导致炼虚境弟子被错误地归类为化神期的掉落范围。

## What Changes
- 修复 `getRarityRangeByRealm` 方法中境界与品阶的映射关系
- 确保炼虚境弟子能获得正确的品阶范围（宝品-地品）
- 将化神期的掉落品阶调整为灵品-宝品

## Impact
- Affected specs: 秘境探索系统、道具掉落系统
- Affected code: `GameEngine.kt` 中的 `getRarityRangeByRealm` 方法

## 问题分析

### 境界数值定义（GameConfig.kt）
```
9 = 炼气
8 = 筑基
7 = 金丹
6 = 元婴
5 = 化神
4 = 炼虚
3 = 合体
2 = 大乘
1 = 渡劫
0 = 仙人
```

### 当前错误的代码逻辑
```kotlin
avgRealm >= 4.0 -> Pair(1, 3)   // 注释说"化神期"，实际匹配炼虚(4)
avgRealm >= 3.0 -> Pair(3, 5)   // 注释说"炼虚期"，实际匹配合体(3)
```

### 问题根源
当 `avgRealm = 4.0`（炼虚期）时，满足 `avgRealm >= 4.0` 条件，返回 `Pair(1, 3)`（凡品-宝品），但根据注释预期，炼虚期应该返回 `Pair(3, 5)`（宝品-地品）。

**条件判断的边界值与注释不匹配！**

### 正确的映射关系
| 境界 | 数值范围 | 应有品阶范围 |
|------|----------|--------------|
| 炼气 | >= 8.0 | 凡品 |
| 筑基 | >= 7.0 且 < 8.0 | 凡品 |
| 金丹 | >= 6.0 且 < 7.0 | 凡品-灵品 |
| 元婴 | >= 5.0 且 < 6.0 | 凡品-灵品 |
| 化神 | >= 4.0 且 < 5.0 | **灵品-宝品** |
| 炼虚 | >= 3.0 且 < 4.0 | 宝品-地品 |
| 合体 | >= 2.0 且 < 3.0 | 宝品-地品 |
| 大乘 | >= 1.0 且 < 2.0 | 宝品-地品 |
| 渡劫 | >= 0.5 且 < 1.0 | 地品-天品 |
| 仙人 | < 0.5 | 地品-天品 |

## ADDED Requirements

### Requirement: 正确的境界品阶映射
系统 SHALL 根据探索队伍的平均境界正确计算掉落道具的品阶范围。

#### Scenario: 炼虚境弟子探索秘境
- **WHEN** 探索队伍平均境界为 4.0（炼虚期）
- **THEN** 掉落道具品阶范围应为宝品-地品（Pair(3, 5)）

#### Scenario: 化神境弟子探索秘境
- **WHEN** 探索队伍平均境界为 5.0（化神期）
- **THEN** 掉落道具品阶范围应为灵品-宝品（Pair(2, 3)）

#### Scenario: 合体境弟子探索秘境
- **WHEN** 探索队伍平均境界为 3.0（合体期）
- **THEN** 掉落道具品阶范围应为宝品-地品（Pair(3, 5)）
