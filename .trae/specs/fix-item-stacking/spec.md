# 修复道具堆叠问题 Spec

## Why
仓库中相同道具（丹药）没有正确堆叠，导致相同名称和品阶的丹药在仓库中显示为多个独立条目，影响用户体验和仓库管理效率。

## What Changes
- 创建统一的 `addPillToWarehouse` 方法，实现丹药堆叠逻辑
- 将所有直接添加丹药到仓库的代码替换为调用新方法

## Impact
- Affected code: `GameEngine.kt` 中所有添加丹药的位置

## ADDED Requirements

### Requirement: 丹药堆叠功能
系统在添加丹药到仓库时，应检查是否已存在相同名称、品阶和类别的丹药，若存在则堆叠数量，否则创建新条目。

#### Scenario: 炼丹成功添加丹药
- **WHEN** 炼丹成功完成
- **THEN** 系统检查仓库中是否存在相同名称、品阶和类别的丹药
- **AND** 若存在则增加数量，否则添加新丹药

#### Scenario: 从商人购买丹药
- **WHEN** 从旅行商人购买丹药
- **THEN** 系统检查仓库中是否存在相同名称、品阶和类别的丹药
- **AND** 若存在则增加数量，否则添加新丹药

#### Scenario: 从宗门商店购买丹药

#### Scenario: 从交易堂下架丹药

- **THEN** 系统检查仓库中是否存在相同名称、品阶和类别的丹药
