# 修复弟子自动服用仙元丹没有增加修为的问题 - 产品需求文档

## 概述
- **摘要**：修复弟子自动服用仙元丹时只增加修炼速度而不增加修为值的问题。
- **目的**：确保弟子自动服用仙元丹时能够正确获得修为增加效果，与手动服用丹药的效果保持一致。
- **目标用户**：游戏玩家

## 目标
- 修复弟子自动服用仙元丹时没有增加修为的问题
- 确保自动服用丹药的效果与手动服用保持一致
- 保持代码的可维护性和一致性

## 非目标（超出范围）
- 不修改其他类型丹药的自动服用逻辑
- 不修改手动服用丹药的逻辑
- 不修改丹药的其他属性或效果

## 背景与上下文
- 仙元丹的效果是立即获得80%修为上限的修为（cultivationPercent = 0.80）
- 目前游戏中弟子自动服用仙元丹时，错误地将修为百分比加成当作了修炼速度加成，没有增加修为值
- 而手动服用仙元丹时，正确地计算并增加了修为值
- 这种不一致导致玩家体验不佳，需要修复

## 功能需求
- **FR-1**：修复 `autoUseCultivationPills` 方法，使其在自动服用仙元丹时同时增加修为值
- **FR-2**：确保自动服用仙元丹的效果与手动服用保持一致

## 非功能需求
- **NFR-1**：修改应保持代码的可维护性和一致性
- **NFR-2**：修改不应影响其他丹药的自动服用逻辑

## 约束
- **技术**：Kotlin语言，Android平台
- **依赖**：现有的游戏引擎架构

## 假设
- 仙元丹属于修炼丹（CULTIVATION）类别
- 自动服用丹药的逻辑在 `GameEngine.kt` 文件中

## 验收标准

### AC-1：自动服用仙元丹增加修为
- **给定**：弟子背包中有仙元丹，且弟子处于修炼状态
- **当**：系统自动服用仙元丹时
- **则**：弟子的修为值应增加，同时修炼速度应提升
- **验证**：`programmatic`

### AC-2：自动服用效果与手动服用一致
- **给定**：同一弟子分别自动和手动服用相同的仙元丹
- **当**：系统处理两种服用方式时
- **则**：两种方式下弟子的修为增加量和修炼速度提升应相同
- **验证**：`programmatic`
