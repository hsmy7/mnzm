# 宗门结盟系统规格

## Why

当前游戏只有简单的好感度系统，玩家与AI宗门之间缺乏深度的外交互动。结盟系统可以增加策略深度，让玩家通过外交手段建立联盟网络，共同对抗强敌或获取资源优势。同时AI宗门之间也会形成动态的联盟关系，创造更丰富的游戏世界。

## What Changes

- 新增玩家与AI宗门结盟功能
- 新增AI宗门之间互相结盟功能
- 新增结盟状态数据结构
- 新增结盟效果系统（资源加成、战争支援、贸易优惠）
- 新增结盟解散机制
- 新增违约机制
- 新增游说系统（派遣弟子进行结盟谈判）
- 新增求援系统（派遣弟子请求盟友支援）
- 修改现有好感度系统，与结盟状态联动

## Impact

- Affected specs: 宗门外交系统、好感度系统、宗门战争系统
- Affected code: GameData.kt, GameEngine.kt, WorldMapGenerator.kt, SectMainScreen.kt, WorldMapScreen.kt

## ADDED Requirements

### Requirement: 结盟数据结构

系统应维护完整的结盟状态数据。

#### Scenario: 结盟记录
- **GIVEN** 两个宗门建立结盟关系
- **WHEN** 结盟成功
- **THEN** 系统创建结盟记录，包含：
  - 结盟双方ID
  - 结盟开始年份
  - 游说弟子ID
  - 结盟有效期：3年

### Requirement: 游说系统

玩家结盟需派遣弟子进行游说谈判。

#### Scenario: 游说条件
- **GIVEN** 玩家向AI宗门发起结盟
- **WHEN** 选择游说弟子
- **THEN** 需满足以下条件：
  - 选择1名空闲状态的弟子
  - 弟子境界需满足目标宗门等级要求：
    - 小型宗门（level=0）：金丹及以上
    - 中型宗门（level=1）：化神及以上
    - 大型宗门（level=2）：炼虚及以上
    - 顶级宗门（level=3）：合体及以上
  - 弟子将被派遣执行游说任务
  - 游说期间弟子状态为忙碌

#### Scenario: 游说成功率计算
- **GIVEN** 玩家派遣弟子游说
- **WHEN** 计算游说成功率
- **THEN** 成功率 = 好感度加成 + 智力加成 + 魅力加成
  - 好感度加成：好感度90以上，每一点加5%（例如好感度95 = 25%加成）
  - 智力加成：以50为基准，每5点加1%，反之减1%
  - 魅力加成：以50为基准，每5点加1%，反之减1%

#### Scenario: 游说成功率示例
- **GIVEN** 游说弟子智力60、魅力55、目标宗门好感度95
- **WHEN** 计算成功率
- **THEN** 成功率 = (95-90)×5% + (60-50)/5×1% + (55-50)/5×1% = 25% + 2% + 1% = 28%

#### Scenario: 游说失败
- **GIVEN** 游说成功率判定失败
- **WHEN** 游说结束
- **THEN** 结盟失败，弟子返回，好感度不变

#### Scenario: 游说成功
- **GIVEN** 游说成功率判定成功
- **WHEN** 游说结束
- **THEN** 结盟成功，扣除结盟费用

### Requirement: 玩家主动结盟

玩家可以向AI宗门发起结盟请求。

#### Scenario: 结盟条件
- **GIVEN** 玩家向AI宗门发起结盟
- **WHEN** 检查结盟条件
- **THEN** 需满足以下条件：
  - 好感度 >= 90
  - 派遣1名空闲弟子进行游说
  - 当前无其他结盟冲突（目标宗门的盟友与玩家敌对）
  - 支付结盟费用（灵石）

#### Scenario: 结盟费用
- **GIVEN** 玩家发起结盟
- **WHEN** 计算结盟费用
- **THEN** 根据目标宗门等级计算：
  - 小型宗门（level=0）: 50,000 灵石
  - 中型宗门（level=1）: 200,000 灵石
  - 大型宗门（level=2）: 800,000 灵石
  - 顶级宗门（level=3）: 2,000,000 灵石

#### Scenario: 结盟被拒
- **GIVEN** 玩家发起结盟
- **WHEN** 条件不满足
- **THEN** 显示拒绝原因，不消耗资源

### Requirement: AI宗门互相结盟

AI宗门之间可以自主建立结盟关系。

#### Scenario: AI结盟触发
- **GIVEN** 年度结算时
- **WHEN** 检查AI宗门关系
- **THEN** 满足以下条件的AI宗门有概率结盟：
  - 双方好感度 >= 90

#### Scenario: AI结盟概率
- **GIVEN** AI宗门满足结盟条件
- **WHEN** 计算结盟概率
- **THEN** 概率 = 基础概率(5%) + 好感度加成(好感度/200)
  - 例如：好感度90，概率 = 5% + 45% = 50%

#### Scenario: AI结盟限制
- **GIVEN** AI宗门尝试结盟
- **WHEN** 检查结盟限制
- **THEN** 每个AI宗门最多同时拥有2个盟友

### Requirement: 结盟效果系统

结盟后双方获得各种效果加成。

#### Scenario: 结盟效果
- **GIVEN** 结盟成功
- **WHEN** 结盟生效
- **THEN** 获得以下效果：
  - 交易价格优惠 10%
  - 可请求战争支援
  - 盟友宗门弟子之间可结为道侣

### Requirement: 跨宗门道侣系统

盟友宗门弟子之间可以结为道侣。

#### Scenario: 跨宗门道侣概率
- **GIVEN** 两个宗门已结盟
- **WHEN** 年度结算时检查弟子关系
- **THEN** 盟友宗门弟子之间有概率结为道侣
  - 概率与门内结合道侣概率一致（基础1.2%）
  - 仅检查成年单身弟子

#### Scenario: 跨宗门生育子嗣
- **GIVEN** 盟友宗门弟子已结为道侣
- **WHEN** 年度结算时检查生育
- **THEN** 有概率生育子嗣
  - 概率与门内生育概率一致（0.6%）
  - 生育后1年内不会再生

#### Scenario: 子嗣归属
- **GIVEN** 跨宗门道侣生育子嗣
- **WHEN** 子嗣出生
- **THEN** 子嗣归属男方宗门

#### Scenario: 解散联盟时子嗣处理
- **GIVEN** 联盟解散
- **WHEN** 存在跨宗门道侣关系
- **THEN** 子嗣归属男方宗门，不随联盟解散而改变

### Requirement: 求援系统

玩家被攻击时可派遣弟子向盟友求援。

#### Scenario: 求援条件
- **GIVEN** 玩家被其他宗门攻击
- **WHEN** 选择求援弟子
- **THEN** 需满足以下条件：
  - 选择1名空闲状态的弟子
  - 弟子境界需金丹及以上
  - 弟子将被派遣执行求援任务
  - 求援期间弟子状态为忙碌

#### Scenario: 求援成功率计算
- **GIVEN** 玩家派遣弟子求援
- **WHEN** 计算求援成功率
- **THEN** 成功率 = 好感度加成 + 智力加成 + 魅力加成
  - 好感度加成：好感度90以上，每一点加5%
  - 智力加成：以50为基准，每5点加1%，反之减1%
  - 魅力加成：以50为基准，每5点加1%，反之减1%

#### Scenario: 求援失败
- **GIVEN** 求援成功率判定失败
- **WHEN** 求援结束
- **THEN** 盟友不派遣支援，弟子返回

#### Scenario: 求援成功
- **GIVEN** 求援成功率判定成功
- **WHEN** 求援结束
- **THEN** 盟友派遣支援队伍

#### Scenario: 支援队伍规模
- **GIVEN** 求援成功
- **WHEN** 计算支援队伍规模
- **THEN** 支援弟子数量 = 5 + (好感度 - 90) × 5
  - 例如：好感度90 = 5名弟子
  - 例如：好感度95 = 30名弟子
  - 例如：好感度100 = 55名弟子（最大）

#### Scenario: 支援队伍境界
- **GIVEN** 求援成功
- **WHEN** 确定支援队伍境界
- **THEN** 根据宗门最高弟子境界确定：
  - 渡劫境宗门（maxRealm=1-2）：派出合体境及以下弟子
  - 大乘境宗门（maxRealm=3）：派出炼虚境及以下弟子
  - 炼虚境宗门（maxRealm=4）：派出化神境及以下弟子
  - 化神境宗门（maxRealm=5）：派出金丹境及以下弟子
  - 金丹境宗门（maxRealm=6）：派出筑基境及以下弟子
  - 其他：比宗门最高境界低一级

#### Scenario: 支援队伍境界补充
- **GIVEN** 支援队伍人数不足
- **WHEN** 指定境界弟子不够
- **THEN** 向上补充更高境界弟子
  - 优先派遣中间力量
  - 弟子不够时派遣更高境界的弟子

### Requirement: 盟约期限机制

盟约有固定期限，到期自动解散。

#### Scenario: 盟约期限
- **GIVEN** 结盟成功
- **WHEN** 盟约生效
- **THEN** 盟约有效期为3年

#### Scenario: 盟约到期
- **GIVEN** 盟约已持续3年
- **WHEN** 年度结算时检查
- **THEN** 自动解除结盟，不扣除好感度

#### Scenario: 盟约续签
- **GIVEN** 盟约即将到期
- **WHEN** 玩家选择续签
- **THEN** 需重新进行游说流程，支付结盟费用

### Requirement: 结盟解散机制

结盟可以被主动或被动解除。

#### Scenario: 主动解除结盟
- **GIVEN** 玩家选择解除结盟
- **WHEN** 确认解除
- **THEN** 产生以下后果：
  - 好感度 -30

#### Scenario: 被动解除结盟
- **GIVEN** 以下情况发生
- **WHEN** 满足条件
- **THEN** 自动解除结盟：
  - 好感度降至90以下
  - 发动战争攻击盟友

#### Scenario: AI解除结盟
- **GIVEN** AI宗门与玩家结盟
- **WHEN** AI决定解除结盟
- **THEN** 好感度 -20

### Requirement: 结盟与战争联动

结盟关系影响战争行为。

#### Scenario: 攻击盟友
- **GIVEN** 玩家攻击自己的盟友
- **WHEN** 发起攻击
- **THEN** 自动解除结盟，盟友好感度降为0

### Requirement: 结盟UI显示

玩家可以查看和管理结盟关系。

#### Scenario: 宗门详情显示
- **GIVEN** 玩家查看宗门详情
- **WHEN** 宗门有结盟关系
- **THEN** 显示：
  - 结盟状态（未结盟/已结盟）
  - 结盟年份
  - 盟约剩余年限
  - 当前好感度
  - 当前效果加成

#### Scenario: 结盟操作按钮
- **GIVEN** 玩家查看未结盟的宗门
- **WHEN** 满足结盟条件
- **THEN** 显示"结盟"按钮
- **WHEN** 已结盟
- **THEN** 显示"解除结盟"按钮

#### Scenario: 游说弟子选择
- **GIVEN** 玩家点击"结盟"
- **WHEN** 显示游说界面
- **THEN** 显示：
  - 可选空闲弟子列表
  - 每个弟子的智力和魅力属性
  - 确认/取消按钮

#### Scenario: 求援弟子选择
- **GIVEN** 玩家被攻击时
- **WHEN** 显示求援界面
- **THEN** 显示：
  - 可选空闲弟子列表
  - 每个弟子的智力和魅力属性
  - 确认/取消按钮

## 核心数值表

### 结盟费用

| 目标宗门等级 | 费用（灵石） |
|------------|------------|
| 小型宗门(0) | 50,000 |
| 中型宗门(1) | 200,000 |
| 大型宗门(2) | 800,000 |
| 顶级宗门(3) | 2,000,000 |

### 游说/求援成功率计算

**公式：** 成功率 = 好感度加成 + 智力加成 + 魅力加成

| 加成类型 | 计算方式 |
|---------|---------|
| 好感度加成 | (好感度 - 90) × 5%，好感度<90时为0% |
| 智力加成 | (智力 - 50) / 5 × 1% |
| 魅力加成 | (魅力 - 50) / 5 × 1% |

**示例：**
- 好感度95、智力60、魅力55
- 成功率 = (95-90)×5% + (60-50)/5×1% + (55-50)/5×1% = 25% + 2% + 1% = 28%

### 结盟条件

| 条件 | 数值 |
|-----|-----|
| 最低好感度 | 90 |
| AI结盟最低好感度 | 90 |
| AI结盟基础概率 | 5% |
| AI结盟好感度加成 | 好感度/200 |
| AI最大盟友数量 | 2 |
| 玩家最大盟友数量 | 3 |

### 游说弟子境界要求

| 目标宗门等级 | 最低境界 |

| 大型宗门(2) | 炼虚 |

### 求援弟子境界要求

| 条件 | 最低境界 |
|-----|---------|

### 盟约期限

| 条件 | 数值 |
|-----|-----|
| 盟约有效期 | 3年 |
| 到期处理 | 自动解散，不扣好感度 |
| 续签条件 | 重新游说，支付费用 |

| 好感度 | 支援弟子数量 |
|-------|------------|
| 90 | 5 |
| 92 | 15 |

**计算公式：** 支援弟子数量 = 5 + (好感度 - 90) × 5

1. 结盟状态显示区域
2. 结盟年份
