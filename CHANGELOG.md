# 模拟宗门 - 更新日志

## [1.4.88] - 2026-03-21

### 修复
- **存档系统安全性修复**
  - 存档哈希算法从MD5升级为SHA-256，提升数据完整性校验安全性
  - 扩展哈希计算覆盖所有重要数据字段（弟子、装备、功法、丹药、材料、灵药、种子、队伍、槽位、联盟等）
  - 修复紧急保存时可能读取不一致状态的问题

- **StateFlow资源优化**
  - 将所有StateFlow的SharingStarted策略从Eagerly改为WhileSubscribed(5000)
  - 减少后台运行时的内存占用和CPU开销

### 优化
- **自动存档配置统一**
  - 自动存档防抖时间统一使用GameConfig配置（30秒）
  - 消除代码中的硬编码配置

- **多版本备份机制**
  - 新增5版本备份轮转机制，保留最近5次存档版本
  - 支持从指定备份版本恢复存档
  - 新增 `getBackupVersions()` 和 `restoreFromBackupVersion()` 方法

- **存档版本迁移**
  - 新增存档版本标识，新存档使用当前游戏版本号
  - 新增版本迁移机制，支持从旧版本存档自动迁移
  - 新增 `migrateSaveData()` 和 `compareVersions()` 方法

- **存档自定义命名**
  - 支持为存档槽位设置自定义名称
  - 新增 `renameSlot()` 方法
  - SaveSlot新增 `customName` 字段和 `displayName` 属性

- **存档健康检查**
  - 新增定期健康检查机制（每60秒）
  - 新增 `performHealthCheck()` 快速检查方法
  - 新增 `performDeepHealthCheck()` 深度检查方法
  - 新增 `repairAllSaves()` 自动修复方法

### 系统
- 版本号：1.4.88 (build 1488)

## [1.4.72] - 2026-03-21

### 优化
- **存档系统全面优化**
  - 压缩策略优化：使用 BEST_SPEED 压缩级别，提升存档速度约 2-3 倍
  - 智能存档防抖：30秒防抖机制 + 数据哈希检测，避免重复存档
  - 异步写入队列：存档操作在后台队列中顺序处理，不阻塞游戏主线程
  - 内存优化：智能 GC 触发策略，仅在内存使用率超过 80% 时触发

### 修复
- **并发安全性修复**
  - 统一锁机制：移除 synchronized 与 Mutex 混用，统一使用 Mutex
  - 防抖变量线程安全：使用 AtomicLong/AtomicReference 替代普通变量
  - 哈希碰撞修复：使用 MD5 哈希包含更多关键字段，降低碰撞概率
  - emergencySave 压缩优化：紧急存档使用与普通存档相同的优化压缩级别

### 新增
- **存档健康检查系统**
  - performHealthCheck()：轻量级快速检查存档健康状态
  - performDeepHealthCheck()：深度检查存档完整性
  - repairAllSaves()：自动修复损坏存档（从备份恢复）
  - shutdown()：SaveManager 生命周期管理方法

- **队列请求保护**
  - onCleared() 中等待队列处理完成（最多5秒）
  - 避免应用退出时队列中的存档请求丢失

### 系统
- 版本号：1.4.72 (build 1472)

## [1.4.44] - 2026-03-17

### 修复
- **存档序列化修复**
  - 修复 Gson 序列化 sealed class 及其子类时的字段冲突问题
  - 问题原因：Kotlin 编译器为 sealed class 生成 `$stable` 内部字段，父类和子类都有该字段导致 Gson 报错
  - 修复方案：配置 Gson 排除以 `$` 开头的 Kotlin 内部字段

### 优化
- **代码重构**
  - 提取统一的 Gson 配置工具类 `GsonConfig`
  - 消除 `SaveManager` 和 `ModelConverters` 中的重复 Gson 配置代码
