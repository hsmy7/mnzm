# 游戏性能优化方案 - 任务计划

## 目标
对 XianxiaSectNative 项目进行4轮循环代码检查，结合国内外权威游戏行业数据（≥20条，近两年），针对设备优化、游戏性能、流畅度、响应速度等玩家体验维度，给出最大化优化方案。

## 阶段

### Phase 1: 项目结构与构建配置检查
- 状态: pending
- 内容: 检查 build.gradle、gradle.properties、ProGuard配置、依赖版本、Baseline Profile等

### Phase 2: 核心业务逻辑与数据层检查
- 状态: pending
- 内容: 检查数据库操作、Repository模式、网络请求、数据序列化、协程使用、内存管理

### Phase 3: UI层与性能热点检查
- 状态: pending
- 内容: 检查Compose重组、图片加载、动画性能、列表性能、启动速度

### Phase 4: 综合复检与遗漏排查
- 状态: pending
- 内容: 跨模块交叉检查、边界条件、线程安全、资源泄漏、已发现问题的二次确认

### Phase 5: 网络研究
- 状态: pending
- 内容: 搜索国内外权威游戏性能优化数据，≥20条参考

### Phase 6: 整合优化方案
- 状态: pending
- 内容: 结合代码检查发现和行业数据，生成最大化优化方案

## 关键约束
- 代码检查不低于4轮复检
- 参考数据不少于20条
- 数据必须是近两年（2024-2026）
- 数据来源必须是国内外权威渠道
