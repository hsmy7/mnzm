## 二、宗门生成算法

### 2.1 核心变更

**旧算法**: 网格抖动 + 力导向松弛
- 将地图划分为 `gridCols x gridRows` 网格
- 每个网格中心放置宗门，叠加 15% 格宽随机抖动
- 力导向松弛 3 次迭代

**新算法**: 聚类生成 + 孤立点 + 力导向 + 重叠解决

```
1. 生成 5~9 个聚类中心 (Cluster)
   - 位置: 地图内部 (margin=400)
   - 半径: 250~700 (随机)
   - 密度: 0.4~1.6 (随机)

2. 按密度权重分配宗门到聚类
   - 聚类内使用圆盘分布: angle ~ U(0, 2π), dist ~ radius * sqrt(U) * U(0.3, 1.0)
   - 位置无效时重试 20 次

3. 5~10 个孤立宗门均匀随机分布
   - 重试 50 次确保最小距离

4. 力导向松弛 (5 次迭代)
   - 理想距离 = sqrt(可用面积 / 宗门数)
   - 边界软约束 (距离 100)

5. 重叠解决 (最多 10 次迭代)
   - 对每对距离 < MIN_SECT_DISTANCE(100) 的宗门双向推离
   - 推离后 clamp 到边界内
```

## 三、路径算法

### 3.1 核心变更

**旧算法**: 直线连接，渲染时基于 hash 随机生成单段二次贝塞尔弯曲
**新算法**: 拓扑层保持直线连接，渲染层生成确定性多航点弯曲路径

### 3.2 航点生成算法

```kotlin
generatePathWaypoints(fromX, fromY, toX, toY, fromId, toId)
  - seed = ((fromId.hashCode * 31 + toId.hashCode) & 0x7FFFFFFFFFFFFFFFL)
  - numWaypoints = PATH_WAYPOINT_MIN + (seed % range)  // 2~4 个
  - 对每个航点 i:
    - t = i / (numWaypoints + 1)  // 均匀分布在线段上
    - base = from + (to - from) * t
    - envelope = sin(t * π)  // 中间大两端小
    - offset = (random(-1, 1)) * distance * PATH_CURVE_STRENGTH * envelope
    - wp = base + normal * offset
    - clamp(wp, 0, MAP_SIZE)
```

### 3.3 渲染算法

使用 **Catmull-Rom 样条近似** 的二次贝塞尔曲线：

```
起点 -> lineTo(起点与第1航点的中点)
航点i -> quadraticTo(航点i, 航点i与航点i+1的中点)
... -> lineTo(终点)
```

### 3.4 交叉处理

连接生成时的交叉惩罚从隐式 300 降到显式 80：

```kotlin
// 旧: neighbor.second + crossingPenalty * 300.0
// 新: neighbor.second + crossingCount * CROSSING_PENALTY(80.0)
```

这意味着交叉不再被强烈排斥，允许路径自然交叉形成路网。

## 四、洞府生成算法

### 4.1 核心变更

**旧算法**: 随机采样 + 点到直线距离检测
**新算法**: 随机采样 + 点到弯曲路径分段距离检测

### 4.2 弯曲路径碰撞检测

```kotlin
isPointNearCurvedPath(px, py, edge, threshold):
  waypoints = generatePathWaypoints(edge.sect1, edge.sect2)
  if waypoints.empty:
    return 点到直线段距离 < threshold
  
  points = [sect1] + waypoints + [sect2]
  for i in 0..points.size-2:
    if 点到线段(points[i], points[i+1])距离 < threshold:
      return true
  return false
```

| 操作 | 规模 | 时间复杂度 | 评估 |

## 九、后续优化建议

1. **航点缓存**: 将 `generatePathWaypoints` 结果缓存到 `MapPathData`，避免 UI 重组时重复计算
2. **存档迁移**: 检测旧存档宗门数量，不足时触发世界重新生成
3. **名称池扩容**: 当前 128 个名称对 80 宗门较紧张，建议扩充到 200+
4. **贝塞尔精确检测**: 洞府碰撞检测可使用贝塞尔曲线采样提高精度
