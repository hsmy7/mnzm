## 一、问题描述

物品卡片左下角错误地显示品阶（如"普通"、"灵品"等）而非品质。需求为：
- 物品卡片左下角应显示品质，而非品阶
- 目前只有丹药有品质（下品/中品/上品），因此只有丹药卡片左下角应显示品质文字
- 其他物品（装备、功法、材料、灵药、种子）的卡片左下角应不显示任何文字
- 品质文字颜色：上品为红色，中品为蓝色，下品为灰色

## 三、详细修改内容

### 3.1 ItemCard.kt

**位置**: `android/app/src/main/java/com/xianxia/sect/ui/components/ItemCard.kt`

**修改1 - 品质文字颜色逻辑**:
```kotlin
// 修改前
if (!data.grade.isNullOrEmpty()) {
    Text(
        text = data.grade,
        fontSize = 9.sp,
        color = rarityColor,  // 使用稀有度颜色
        ...
    )
}

// 修改后
if (!data.grade.isNullOrEmpty()) {
    Text(
        text = data.grade,
        fontSize = 9.sp,
        color = getQualityColor(data.grade),  // 使用品质专属颜色
        ...
    )
}
```

**修改2 - 新增品质颜色函数**:
```kotlin
fun getQualityColor(quality: String?): Color = when (quality) {
    "上品" -> Color(0xFFE74C3C)  // 红色
    "中品" -> Color(0xFF3498DB)  // 蓝色
    "下品" -> Color(0xFF95A5A6)  // 灰色
    else -> Color.Transparent
}
```

```kotlin
// 移除: grade = getRarityName(item.rarity)
```

3. **功法学习选择对话框** (行 ~1004):
```kotlin
// 移除: grade = getRarityName(stack.rarity)
```

4. **移除未使用的 import**:

### 3.3 AlchemyScreen.kt

**位置**: `android/app/src/main/java/com/xianxia/sect/ui/game/AlchemyScreen.kt`

### 3.4 MainGameScreen.kt

**位置**: `android/app/src/main/java/com/xianxia/sect/ui/game/MainGameScreen.kt`

**问题**: 仓库界面存在预先未提交的代码，导致 `selectedItem?.let` 块中 Composable 调用上下文错误，以及类型推断失败。

**修复**:
1. `selectedItem` 改为使用 `derivedStateOf` 包装，解决类型推断问题
2. `selectedItem?.let { item -> ... }` 改为 `if (showDetailDialog && selectedItem != null) { val item = selectedItem!! ... }`，解决 Composable 上下文错误
