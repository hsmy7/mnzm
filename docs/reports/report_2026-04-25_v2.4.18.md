## 一、问题描述

弟子自动使用功能（自动使用丹药、自动装备、自动学习功法）的判定时机为每月一次，导致弟子获得新物品后需要等待到下一个月份才能自动使用，响应不够及时。需求为将判定时机从每月判定改为每日判定。

### 3.2 冷却期精度提升

**文件**: `StorageBagUtils.kt`

**修改前**: 冷却期 = 3个月，基于 `forgetYear * 12 + forgetMonth` 计算

**修改后**: 冷却期 = 90天，基于 `forgetYear * 360 + (forgetMonth - 1) * 30 + forgetDay` 计算

**旧存档兼容**: `forgetDay` 为 null 时回退到月度计算

```kotlin
fun isInCoolingPeriod(item: StorageBagItem, currentYear: Int, currentMonth: Int, currentDay: Int): Boolean {
    val forgetYear = item.forgetYear ?: return false
    val forgetMonth = item.forgetMonth ?: return false
    val forgetDay = item.forgetDay
    if (forgetDay != null) {
        val forgetTotalDays = forgetYear * 360 + (forgetMonth - 1) * 30 + forgetDay
        val currentTotalDays = currentYear * 360 + (currentMonth - 1) * 30 + currentDay
        return currentTotalDays - forgetTotalDays < COOLING_PERIOD_DAYS
    } else {
        // 旧存档兼容：回退到月度计算
        val forgetTotalMonths = forgetYear * 12 + forgetMonth
        val currentTotalMonths = currentYear * 12 + currentMonth
        return currentTotalMonths - forgetTotalMonths < COOLING_PERIOD_MONTHS
    }
}
```

### 3.7 旧存档兼容性

**文件**: `SerializationModule.kt` (`convertBackDisciple`)

旧存档加载时，`cultivationSpeedDuration` 和 `pillEffectDuration` 的值是月度的（通常 ≤ 12），需要乘以30转换为日度：

```kotlin
cultivationSpeedDuration = if (data.cultivationSpeedDuration > 0 && data.cultivationSpeedDuration <= 12)
    data.cultivationSpeedDuration * 30 else data.cultivationSpeedDuration

pillEffectDuration = if (data.pillEffectDuration > 0 && data.pillEffectDuration <= 12)
    data.pillEffectDuration * 30 else data.pillEffectDuration
```

判断条件 `<= 12` 用于区分旧存档（月度值）和新存档（日度值），避免重复转换。
