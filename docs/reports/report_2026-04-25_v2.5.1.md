## 二、关系等级系统

### 2.1 等级定义

```kotlin
enum class SectRelationLevel(
    val displayName: String,
    val minFavor: Int,
    val maxFavor: Int,
    val colorHex: Long
) {
    HOSTILE("敌对", 0, 9, 0xFFF44336),
    ANTAGONISTIC("交恶", 10, 39, 0xFFFF5722),
    NORMAL("普通", 40, 59, 0xFFFF9800),
    FRIENDLY("友善", 60, 79, 0xFF8BC34A),
    INTIMATE("至交", 80, 100, 0xFF4CAF50);

    val maxAllowedRarity: Int
        get() = when (this) {
            NORMAL -> 2    // 灵品
            FRIENDLY -> 4  // 玄品
            INTIMATE -> 6  // 天品
            else -> 0      // 不可交易
        }
}
```

### 2.2 等级与功能对应

| 等级 | 好感度 | 可攻击 | 可交易 | 可结盟 | 可购买品质 |

| 敌对 | 0-9 | 是 | 否 | 否 | 不可交易 |
| 交恶 | 10-39 | 否 | 否 | 否 | 不可交易 |
| 普通 | 40-59 | 否 | 是 | 否 | 灵品及以下(rarity≤2) |
| 友善 | 60-79 | 否 | 是 | 否 | 玄品及以下(rarity≤4) |
| 至交 | 80-100 | 否 | 是 | 是 | 所有物品(rarity≤6) |

## 三、宗门送礼系统

### 3.1 核心变更

**旧系统**: 双Tab送礼（灵石送礼 + 物品送礼）
- 物品送礼支持功法、装备、丹药三类
- 每种物品有稀有度，影响好感度增长和拒绝概率
- 宗门有物品偏好类型（装备/功法/丹药/灵石/无）

**新系统**: 单Tab灵石送礼
- 仅支持灵石送礼，4个档位（薄礼/厚礼/重礼/大礼）
- 移除物品送礼UI和相关参数传递
- 保留灵石偏好机制（偏好灵石时好感度1.3倍，拒绝概率-15%）

### 3.2 移除的组件

### 3.3 保留的偏好机制

虽然UI不再显示物品偏好，但灵石偏好仍然有效：

```kotlin
// 偏好灵石时：好感度增长 x1.3，拒绝概率 -15%
private fun calculatePreferenceMultiplier(...): Double {
    return when {
        isSpiritStone && giftPreference == GiftPreferenceType.SPIRIT_STONE -> 1.3
        else -> 1.0
    }
}
```

## 四、宗门交易系统

### 4.1 核心变更

**旧系统**: 好感度线性解锁
- 好感度>=40才能交易
- 好感度每增加10点，解锁更高一级稀有度
- 好感度>=70享受折扣

**新系统**: 关系等级阶梯解锁
- 普通/友善/至交关系才能交易
- 每个等级对应固定的最大可购买稀有度
- 折扣阈值保持好感度>=70（与FRIENDLY等级部分重叠）

### 4.2 交易验证逻辑

```kotlin
// EventService.validateSectTrade()
val relationLevel = GameUtils.getSectRelationLevel(relation)

// 1. 检查交易权限
if (relationLevel !in listOf(NORMAL, FRIENDLY, INTIMATE)) {
    addGameEvent("关系不足，无法与${sect.name}交易", EventType.WARNING)
    return null
}

// 2. 检查物品品质限制
val maxAllowedRarity = relationLevel.maxAllowedRarity
if (item.rarity > maxAllowedRarity) {
    addGameEvent("关系等级不足，无法购买${GameConfig.Rarity.get(item.rarity).name}物品${item.name}", EventType.WARNING)
    return null
}
```

### 4.3 UI显示

交易对话框头部显示：
```
关系: 友善(65) (盟友) (3.0%折扣)
```

不可购买的商品显示为灰色，点击时提示"关系不足，无法交易该物品"。

## 九、后续优化建议

1. **折扣阈值统一**: 当前折扣阈值仍使用硬编码的70，建议纳入SectRelationLevel体系
2. **GiftPreferenceType清理**: 如果确定不再恢复物品送礼，可考虑移除EQUIPMENT/MANUAL/PILL枚举值（需处理存档迁移）
3. **关系等级事件**: 可考虑在关系等级变化时触发特殊事件（如从交恶变为普通时解锁交易）
4. **AI外交行为**: 可根据关系等级调整AI宗门的外交行为（如敌对宗门更可能发动攻击）
