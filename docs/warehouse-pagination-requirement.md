# 宗门仓库分页功能需求文档

## 一、需求背景

### 当前状态
- 仓库UI采用 `LazyVerticalGrid` 布局，固定6列
- 所有物品一次性加载展示，无分页机制
- 已有筛选功能（全部/装备/丹药/功法/草药/种子/材料）

### 问题
- 物品数量增长后，长列表滚动体验下降
- 用户难以快速定位特定物品
- 内存占用随物品增长而增加

## 二、需求目标

实现仓库分页展示，每页固定显示 **12行 × 6列 = 72个物品槽位**。

## 三、功能规格

### 3.1 布局规格

| 参数 | 数值 | 说明 |
|------|------|------|
| 每页行数 | 12行 | 固定 |
| 每页列数 | 6列 | 固定（与现有保持一致） |
| 每页物品数 | 72个 | 12 × 6 |
| 物品间距 | 4dp | 与现有保持一致 |

### 3.2 分页控件

**位置**：仓库列表底部

**组成**：
- 上一页按钮（首页时禁用）
- 页码显示：`当前页/总页数`（如：`3/10`）
- 下一页按钮（末页时禁用）
- 快速跳转：首页/末页按钮（可选）

### 3.3 交互规则

1. **翻页行为**
   - 点击上/下一页，切换到相邻页
   - 翻页保持当前筛选条件

2. **筛选联动**
   - 切换筛选条件时，自动回到第1页
   - 重新计算总页数

3. **物品操作**
   - 物品详情弹窗关闭后，保持当前页码
   - 出售物品后，刷新当前页数据

### 3.4 边界情况

| 场景 | 处理方式 |
|------|----------|
| 物品总数 < 72 | 显示1页，不足部分留空 |
| 物品总数 = 0 | 显示空状态提示 |
| 筛选后无物品 | 显示空状态，页码显示 `0/0` |
| 最后1页物品被全部出售 | 自动跳转到前一页 |

## 四、技术方案

### 4.1 数据层

**已有基础设施可复用**：
- `WarehousePager.loadPage()` - 分页加载逻辑
- `WarehousePage` 数据模型 - 包含分页元数据
- `SectWarehouseManager.loadPage()` - 管理器接口

**需调整**：
- `DEFAULT_PAGE_SIZE` 从 20 改为 72

**相关文件**：
- `core/warehouse/WarehousePager.kt`
- `core/warehouse/WarehouseModels.kt`
- `core/engine/SectWarehouseManager.kt`

### 4.2 UI层

**改造 `WarehouseTab` 组件**：
- 引入 `currentPage` 状态
- 底部添加分页控件
- 物品列表高度固定（12行高度）

**相关文件**：
- `ui/game/MainGameScreen.kt` - `WarehouseTab` 组件（约第3712行起）

### 4.3 实现步骤

1. 修改 `WarehousePager.DEFAULT_PAGE_SIZE` 为 72
2. 在 `WarehouseTab` 中添加分页状态管理
3. 实现分页控件UI组件
4. 实现翻页逻辑与筛选联动
5. 处理边界情况

## 六、验收标准

1. 每页显示72个物品槽位（12行×6列）
2. 分页控件功能正常
3. 筛选与分页联动正确
4. 翻页流畅，无明显卡顿
5. 边界情况处理正确

## 七、风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 物品数量极少时分页控件占位 | 低 | 物品少于72时隐藏分页控件 |
| 筛选切换频繁导致页码闪烁 | 低 | 添加状态防抖处理 |
| 出售物品后页码计算错误 | 中 | 出售后重新计算总页数并校正当前页 |

## 八、后续优化方向

- 添加页码快速跳转输入框
- 支持每页显示数量自定义
- 添加物品拖拽排序功能
- 支持多选批量操作
