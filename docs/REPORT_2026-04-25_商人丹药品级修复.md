# 商人丹药品级刷新修复报告

**版本**：v2.5.8 → v2.5.10
**日期**：2026-04-25
**涉及文件**：`CultivationService.kt`、`build.gradle`、`CHANGELOG.md`

---

## 一、问题描述

商人界面（云游商人）刷新丹药时，**所有丹药只显示"上品"品质**，中品和下品丹药从未出现。

### 1.1 预期行为
- 商人应刷新各稀有度（凡品~天品）的丹药
- 每种丹药应有不同品质（下品/中品/上品）出现

### 1.2 实际行为
- 商人刷新的丹药**全部是上品**
- 中品和下品丹药被覆盖，无法出现

---

## 二、根因分析

### 2.1 数据层：同名丹药多品质模板

`ItemDatabase.allPills` 中，每种丹药按 `tier × grade` 生成模板。以"引灵丹"为例：

| id | name | grade | rarity | price |
|---|---|---|---|---|
| `cultivationSpeed_1_low` | 引灵丹 | LOW（下品） | 1 | 50 |
| `cultivationSpeed_1_medium` | 引灵丹 | MEDIUM（中品） | 1 | 100 |
| `cultivationSpeed_1_high` | 引灵丹 | HIGH（上品） | 1 | 200 |

同名丹药有 **3 个品质模板**，`name` 相同但 `grade` 和 `price` 不同。

### 2.2 池构建：Map 覆盖导致只保留上品

`buildMerchantItemPools()` 原逻辑：

```kotlin
ItemDatabase.allPills.values.forEach { t ->
    pools.poolByRarity.getOrPut(t.rarity) { mutableListOf() }.add(t.name to "pill")
    pools.rarityMap[t.name] = t.rarity
    pools.priceMap[t.name] = (t.price * ...).roundToInt().toLong()
    pools.gradeMap[t.name] = t.grade.displayName  // 同名覆盖！
}
```

由于 `gradeMap`、`priceMap`、`rarityMap` 均以**丹药名**为键，同名不同品质的 3 个模板遍历时**后写入覆盖前者**。`PillGrade.entries` 顺序为 LOW → MEDIUM → HIGH，最终只保留 **HIGH（上品）**。

### 2.3 合并逻辑：不同品质被错误合并

`mergeMerchantItems()` 原合并键：

```kotlin
val key = "${item.name}:${item.type}"
```

不含品质维度。同名不同品质的丹药被错误合并为一条，数量叠加、价格取加权平均，品质只保留第一个出现的。

### 2.4 空池崩溃风险

`selectItemByRarity()` 和 `selectFirstAvailableItem()` 中：

```kotlin
itemPoolByRarity[rarity]?.random()
```

若某稀有度列表为空（非 null），`.random()` 会抛出 `NoSuchElementException`，fallback 逻辑永远不会执行。

---

## 三、修复方案

### 3.1 v2.5.8：修复覆盖与合并问题

#### (1) 引入 `PoolEntry` 数据类

替代 `Pair<String, String>`，携带 `name`/`type`/`grade`，提供 `compositeKey`（`name|grade`）：

```kotlin
private data class PoolEntry(
    val name: String,
    val type: String,
    val grade: String? = null
) {
    val compositeKey: String get() = if (grade != null) "$name|$grade" else name
}
```

#### (2) 丹药使用复合键注册

```kotlin
ItemDatabase.allPills.values.forEach { t ->
    val gradeName = t.grade.displayName
    val entry = PoolEntry(t.name, "pill", gradeName)
    pools.poolByRarity.getOrPut(t.rarity) { mutableListOf() }.add(entry)
    pools.rarityMap[entry.compositeKey] = t.rarity      // "引灵丹|下品"
    pools.priceMap[entry.compositeKey] = (t.price * ...).toLong()
    pools.gradeMap[entry.compositeKey] = gradeName
}
```

同名不同品质丹药使用不同的复合键，不再互相覆盖。

#### (3) 合并键加入品质维度

```kotlin
val key = if (item.grade != null)
    "${item.name}:${item.type}:${item.grade}"
else
    "${item.name}:${item.type}"
```

不同品质丹药独立存在，不再错误合并。

#### (4) 空列表安全

```kotlin
itemPoolByRarity[rarity]?.takeIf { it.isNotEmpty() }?.random()
```

空列表时返回 `null`，触发 fallback 逻辑。

### 3.2 v2.5.10：调整品级概率分布

#### (1) 问题：等权概率不合理

v2.5.8 修复后，每个丹药名在池中有 3 个等权条目（下品/中品/上品各 1 个），选中概率均为 **1/3**。这与设计预期不符。

#### (2) 方案：单条目 + 独立概率抽取

- **池构建**：每个丹药名只添加 **1 个条目**（以中品为基准价格）
- **品级抽取**：创建商品时按指定概率独立抽取品级
- **价格调整**：按品级倍率动态计算

```kotlin
private fun selectMerchantPillGrade(): PillGrade {
    val roll = Random.nextDouble()
    return when {
        roll < 0.03 -> PillGrade.HIGH    // 上品 3%
        roll < 0.40 -> PillGrade.MEDIUM  // 中品 37%
        else -> PillGrade.LOW             // 下品 60%
    }
}
```

```kotlin
val grade: PillGrade? = if (entry.type == "pill") selectMerchantPillGrade() else null
val adjustedPrice = if (grade != null)
    (basePrice * grade.priceMultiplier / PillGrade.MEDIUM.priceMultiplier).roundToLong()
else
    basePrice
```

#### (3) 简化 `PoolEntry`

品级不再在池构建阶段参与，`PoolEntry` 简化为：

```kotlin
private data class PoolEntry(val name: String, val type: String)
```

移除 `gradeMap` 和 `compositeKey`。
