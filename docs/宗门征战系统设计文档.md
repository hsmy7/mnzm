## 一、系统概述

### 1.1 设计目标
参照文明六的征战模式，实现玩家组建军团进攻AI宗门的核心玩法。胜利后可占领AI宗门，获得灵矿资源。

### 1.2 核心玩法循环
```
组建军团(10人) → 选择目标 → 行军(5秒/节点) → 战斗(10v10) → 占领 → 采矿收益
```

## 三、核心系统规则

### 3.1 军团组建规则

| 项目 | 规则 |
|------|------|
| 队伍上限 | 无限制 |
| 队伍规模 | **固定10人**（必须满员） |
| 创建消耗 | 无 |
| 月度消耗 | **5000灵石/月** |
| 弟子状态 | 必须存活且空闲 |

### 3.2 驻扎规则

| 规则 | 说明 |
|------|------|
| 单军团限制 | **每个宗门只能驻扎1只军团** |
| 玩家宗门 | 可驻扎1只军团 |
| 占领地 | 占领军团自动驻扎 |
| 冲突处理 | 进攻方胜利后，原驻扎军团强制返回 |

### 3.3 行军规则

| 项目 | 规则 |
|------|------|
| 时间计算 | **5秒/节点**（相邻宗门） |
| 路线显示 | 在WorldMap上显示路线和军团名称 |
| 名称移动 | 军团名称随行军进度在路线上移动 |
| 路径计算 | 使用BFS算法寻找最短路径 |

### 3.4 战斗规则

| 项目 | 规则 |
|------|------|
| 战斗规模 | **10v10团队战** |
| 战斗方式 | 回合制，按速度排序行动 |
| 最大回合 | 50回合 |
| 超时判定 | 剩余总血量多者胜 |
| 伤亡系统 | 轻伤/中伤/重伤/死亡 |
| 突破限制 | **战斗中弟子无法突破境界** |

### 3.5 防守队伍规则

| 场景 | 防守队伍来源 |
|------|-------------|
| 未被占领AI宗门 | 根据disciples数据生成10名AI弟子 |
| 已被占领宗门 | **驻扎军团作为防守队伍** |
| AI属性计算 | 使用游戏内统一属性公式 |

### 3.6 占领后灵矿规则

| 项目 | 规则 |
|------|------|
| 槽位数量 | **3个采矿槽位** |
| 采矿逻辑 | 复用现有MineSlot系统 |
| 产出计算 | 根据宗门等级（小型100/中型200/大型350/顶级500） |
| 结算方式 | 与玩家宗门灵矿一同月度结算 |

## 四、核心流程设计

### 4.1 创建军团流程

```
1. 玩家选择10名弟子
   ↓
2. 验证弟子状态（存活+空闲）
   ↓
3. 输入军团名称
   ↓
4. 设置弟子状态为WAR
   ↓
5. 军团创建完成，驻扎在玩家宗门
```

### 4.2 出征流程

```
1. 选择出征军团
   ↓
2. 选择目标宗门（只能进攻相邻的非己方宗门）
   ↓
3. 计算行军路线（BFS）
   ↓
4. 确定防守方
   ├─ 目标已被占领 → 驻扎军团作为防守方
   └─ 目标未被占领 → 生成AI防守队伍
   ↓
5. 开始行军（每节点5秒）
   ↓
6. 到达后触发战斗
   ↓
7. 判定胜负
   ├─ 胜利 → 占领宗门，原防守军团（如有）被击败返回
   └─ 失败 → 军团返回
```

### 4.3 战斗流程

```
1. 获取双方10名弟子
   ↓
2. 转换为Combatant
   ↓
3. 按速度排序
   ↓
4. 循环执行回合（最多50回合）
   ├─ 每个存活单位行动一次
   ├─ 随机选择目标攻击
   └─ 记录战斗日志
   ↓
5. 检查胜负条件
   ├─ 一方全灭 → 结束战斗
   ├─ 达到50回合 → 血量多者胜
   └─ 继续下一回合
   ↓
6. 计算伤亡
   ↓
7. 返回战斗结果
```

### 4.4 占领流程

```
战斗胜利
   ↓
处理原防守方（如果目标已被占领）
   ├─ 有驻扎军团 → 该军团被击败，强制返回玩家宗门
   └─ 无驻扎军团 → 无需处理
   ↓
设置占领状态
   ├─ sect.isOccupied = true
   ├─ sect.occupierTeamId = team.id
   ├─ sect.occupierTeamName = team.name
   └─ 清空mineSlots
   ↓
更新进攻军团状态
   ├─ team.stationedSectId = sect.id
   ├─ team.stationedSectName = sect.name
   └─ team.status = IDLE
   ↓
发送占领通知
```

### 4.5 月度结算流程

```
1. 计算所有军团月度消耗
   ↓
2. 检查灵石是否充足
   ├─ 充足 → 扣除灵石
   └─ 不足 → 军团解散
       ├─ 弟子状态恢复IDLE
       ├─ 占领地失去防守（放弃占领）
       └─ 删除军团数据
   ↓
3. 结算占领地灵矿产出
   ↓
4. 发放资源到玩家
```

## 五、关键算法

### 5.1 行军路线计算（BFS）

```kotlin
fun calculatePath(fromSectId: String, toSectId: String): List<String> {
    val visited = mutableSetOf<String>()
    val queue = ArrayDeque<Pair<String, List<String>>>()
    queue.add(fromSectId to listOf(fromSectId))
    
    while (queue.isNotEmpty()) {
        val (current, path) = queue.removeFirst()
        if (current == toSectId) return path
        
        val sect = getWorldSect(current)
        for (neighborId in sect.connectedSectIds) {
            if (neighborId !in visited) {
                visited.add(neighborId)
                queue.add(neighborId to path + neighborId)
            }
        }
    }
    return emptyList() // 无路径
}
```

### 5.2 AI防守队伍生成

```kotlin
fun generateAIDefenseTeam(sect: WorldSect): List<DefenseDisciple> {
    val team = mutableListOf<DefenseDisciple>()
    val sectDisciples = sect.disciples // Map<realm, count>
    
    // 从高境界开始选取
    var remaining = 10
    for (realm in 0..9) {
        if (remaining <= 0) break
        val count = sectDisciples[realm] ?: 0
        val takeCount = minOf(count, remaining)
        
        repeat(takeCount) {
            team.add(createDefenseDisciple(sect, realm))
        }
        remaining -= takeCount
    }
    
    // 不足10人则填充
    while (team.size < 10) {
        val highestRealm = (0..9).firstOrNull { (sectDisciples[it] ?: 0) > 0 } ?: 9
        team.add(createDefenseDisciple(sect, highestRealm))
    }
    
    return team.take(10)
}

fun createDefenseDisciple(sect: WorldSect, realm: Int): DefenseDisciple {
    val realmConfig = GameConfig.Realm.get(realm)
    val layer = Random.nextInt(1, 10)
    val layerBonus = 1.0 + (layer - 1) * 0.1
    
    return DefenseDisciple(
        name = generateDefenseName(sect.name),
        realm = realm,
        realmLayer = layer,
        realmName = realmConfig.name,
        maxHp = (100 * realmConfig.multiplier * layerBonus).toInt(),
        maxMp = (50 * realmConfig.multiplier * layerBonus).toInt(),
        physicalAttack = (10 * realmConfig.multiplier * layerBonus).toInt(),
        magicAttack = (5 * realmConfig.multiplier * layerBonus).toInt(),
        physicalDefense = (5 * realmConfig.multiplier * layerBonus).toInt(),
        magicDefense = (3 * realmConfig.multiplier * layerBonus).toInt(),
        speed = (10 * realmConfig.multiplier * layerBonus).toInt(),
        critRate = 0.05
    )
}
```

### 5.3 战斗回合执行

```kotlin
fun executeBattleRound(
    playerTeam: List<Combatant>,
    defenseTeam: List<Combatant>,
    roundNum: Int
): BattleRoundLog {
    val actions = mutableListOf<BattleAction>()
    
    // 按速度排序所有存活单位
    val allUnits = (playerTeam + defenseTeam)
        .filter { !it.isDead }
        .sortedByDescending { it.speed }
    
    for (unit in allUnits) {
        if (unit.isDead) continue
        
        // 选择目标（敌方存活单位）
        val targets = if (unit.type == CombatantType.DISCIPLE) 
            defenseTeam else playerTeam
        val target = targets.filter { !it.isDead }.randomOrNull() ?: continue
        
        // 执行攻击
        val action = executeAttack(unit, target)
        actions.add(action)
    }
    
    return BattleRoundLog(round = roundNum, actions = actions)
}
```

### 5.4 占领地灵矿产出计算

```kotlin
fun calculateOccupiedMineOutput(sectLevel: Int, activeSlots: Int): Long {
    val baseOutput = when (sectLevel) {
        0 -> 100L   // 小型
        1 -> 200L   // 中型
        2 -> 350L   // 大型
        3 -> 500L   // 顶级
        else -> 100L
    }
    return baseOutput * activeSlots / 3
}

## 附录：属性计算公式

### AI防守弟子属性

```
基础值 × 境界倍率 × 层数加成

其中：
- 基础值：HP=100, MP=50, 物攻=10, 法攻=5, 物防=5, 法防=3, 速度=10

- 层数加成：1.0 + (层数-1) × 0.1
