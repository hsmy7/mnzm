# 定向推送/广告合规问题分析报告

**应用名称**：模拟宗门  
**应用包名**：com.xianxia.sect  
**当前版本**：2.5.16（versionCode 2084）  
**报告日期**：2026年4月25日  
**分析范围**：Android 端完整代码库（`android/app/src/main`）

---

## 一、问题概述

本次分析依据《APP违法违规收集使用个人信息行为认定方法》、工信部关于APP侵害用户权益专项整治工作要求，重点检查以下行为：

- APP、SDK未以显著方式标示且未经用户同意，将收集到的用户搜索、浏览记录、使用习惯等个人信息用于定向推送或广告精准营销
- 未提供关闭定向推送/广告功能选项

## 二、核心发现

### 2.1 应用本身无定向推送/广告功能

经全面扫描代码库（`build.gradle` 依赖、AndroidManifest.xml、所有 Kotlin/Java 源文件），**应用自身未集成任何广告SDK或推送SDK**，也未实现任何个性化推荐、定向推送、精准营销功能。

| 检查项 | 结果 |
|--------|------|
| 广告SDK（AdMob、穿山甲/Pangle、广点通/GDT、Unity Ads、Facebook Audience Network、百度广告、快手广告等） | 未集成 |
| 推送SDK（FCM、极光推送/JPush、个推、友盟推送、OneSignal等） | 未集成 |
| 用户行为追踪/统计SDK（友盟统计/Umeng、TalkingData、神策数据、GrowingIO、Firebase Analytics等） | 未集成 |
| 应用内定向推送功能 | 未实现 |
| 应用内个性化广告/推荐功能 | 未实现 |

### 2.2 关键风险点：TapTap SDK 的 OAID 收集与"广告效果分析"

应用集成了 **TapTap SDK（v4.10.0）**，该 SDK 存在与定向广告相关的数据收集行为。

#### 证据一：隐私政策中的 OAID 说明

文件：`android/app/src/main/java/com/xianxia/sect/ui/PrivacyConsentScreen.kt`

> "TapTap SDK也会独立收集 Android ID 和 **OAID**（开放匿名设备标识符），用于设备系统兼容性、定位解决问题和**广告效果分析**。"

#### 证据二：完整隐私政策中的 SDK 数据收集说明

> "tap-core：SDK核心功能 — 可能收集设备信息（设备型号、操作系统版本、Android ID、**OAID**、CPU信息、内存信息）、网络信息（网络类型）"

#### 证据三：build.gradle 中的 TapTap SDK 依赖

文件：`android/app/build.gradle`

```gradle
// TapTap SDK
implementation 'com.taptap.sdk:tap-login:4.10.0'
implementation 'com.taptap.sdk:tap-core:4.10.0'
implementation 'com.taptap.sdk:tap-common:4.10.0'
implementation 'com.taptap.sdk:tap-compliance:4.10.0'
```

#### 证据四：AndroidManifest.xml 中禁用自动初始化

文件：`android/app/src/main/AndroidManifest.xml`

```xml
<!-- 禁用 TapTap SDK 的 ContentProvider 自动初始化，确保在用户同意隐私政策后才初始化 SDK -->
<provider
    android:name="com.taptap.sdk.kit.internal.TapTapKitInitProvider"
    android:authorities="${applicationId}.TapTapKitInitProvider"
    android:exported="false"
    tools:node="remove"
    tools:ignore="MissingClass" />
```

## 三、合规缺陷详细分析

### 缺陷一：OAID 用途未以显著方式单独标示

| 项目 | 说明 |
|------|------|
| **问题描述** | 隐私政策中虽然提到了"广告效果分析"，但OAID作为专门用于广告追踪的标识符，其收集目的未以**显著方式**单独标示。OAID（Open Anonymous Device Identifier）是中国广告协会推出的专门用于广告场景的设备标识符，与普通的设备信息有本质区别。 |
| **涉及文件** | `PrivacyConsentScreen.kt`（第249行、第564行、第610行） |
| **法规依据** | 《APP违法违规收集使用个人信息行为认定方法》第三条：未明示收集使用个人信息的目的、方式和范围 |
| **风险等级** | 中 |

### 缺陷二：缺乏关闭 OAID/广告追踪的选项

| 项目 | 说明 |
|------|------|
| **问题描述** | 应用中**没有任何设置项或开关**允许用户关闭 TapTap SDK 的 OAID 收集或广告追踪功能。虽然应用本身没有广告，但 TapTap SDK 收集 OAID 用于"广告效果分析"，用户无法拒绝或关闭这一行为。 |
| **涉及文件** | 全局（应用设置中无相关选项） |
| **法规依据** | 工信部《关于开展APP侵害用户权益专项整治工作的通知》：重点整治未提供关闭定向推送功能选项的行为 |
| **风险等级** | **高** |

### 缺陷三：隐私政策表述模糊

| 项目 | 说明 |
|------|------|
| **问题描述** | "广告效果分析"的具体含义未向用户解释清楚，用户无法理解该数据收集是否与定向广告、精准营销相关。 |
| **涉及文件** | `PrivacyConsentScreen.kt` |
| **法规依据** | 《个人信息保护法》第十七条：个人信息处理者在处理个人信息前，应当以显著方式、清晰易懂的语言真实、准确、完整地向个人告知相关事项 |
| **风险等级** | 中 |

## 四、OAID 的特殊敏感性说明

OAID（开放匿名设备标识符）是中国信息通信研究院推出的**广告专用设备标识符**，其设计初衷就是为了替代不可重置的 IMEI，用于广告追踪和效果归因。根据工信部相关整治要求：

1. 收集 OAID 必须**明确告知用户其用于广告目的**
2. 必须提供**关闭或重置**的选项
3. 不得将 OAID 与用户的其他个人信息关联用于定向推送

---

## 五、隐私同意机制评估

应用在 `MainActivity.kt`（第147行）中实现了首次启动的隐私政策弹窗：

- 用户必须点击"同意"才能继续使用
- 点击"不同意"则退出应用
- TapTap SDK 仅在用户同意后才初始化

**评估结论**：同意机制本身设计合理，符合"未经用户同意不得收集个人信息"的要求。但同意内容中对 OAID 的广告用途标示不够显著，且用户同意后将被迫接受 OAID 的不可逆收集（无关闭选项）。

## 六、整改建议

### 建议一：显著标示 OAID 用途（优先级：高）

在隐私政策中单独、显著地说明 OAID 的收集目的，使用加粗、高亮、单独段落等方式与普通设备信息区分开。

**示例修改**：

> **【广告标识符收集特别提示】**  
> TapTap SDK 会收集您的 **OAID（开放匿名设备标识符）**。OAID 是专门用于广告场景的设备标识符，TapTap 使用 OAID 进行**广告效果分析**（包括广告展示、点击、转化等数据的归因统计）。该标识符与您的 TapTap 账户信息结合后，可能用于优化您在 TapTap 平台上的广告体验。

### 建议二：增加 OAID/广告追踪关闭选项（优先级：高）

在应用设置中增加开关，允许用户关闭 TapTap SDK 的 OAID 收集。

**实现思路**：
1. 在设置页面新增"隐私设置"或"广告偏好"入口
2. 提供"限制广告追踪"开关
3. 开关状态保存到本地存储
4. 在初始化 TapTap SDK 前读取该开关状态，如用户选择关闭，则通过 TapTap SDK 提供的隐私配置接口限制 OAID 收集（需查阅 TapTap SDK v4.10.0 文档确认具体接口）

### 建议三：更新隐私政策文本（优先级：中）

明确告知用户：
- OAID 是什么
- 收集 OAID 的具体目的（广告效果分析的具体含义）
- 用户如何关闭该收集（配合建议二）

### 建议四：考虑移除 OAID 收集（优先级：低）

如果应用本身没有广告业务，可与 TapTap 确认是否可以在初始化 SDK 时禁用 OAID 收集，或仅保留防沉迷合规所需的最小数据收集。

## 七、总结

| 判定项 | 结论 |
|--------|------|
| 应用是否存在强制用户使用定向推送功能 | **不存在**（应用本身无此功能） |
| 应用是否未经用户同意将个人信息用于定向推送/广告 | **存在间接风险**（通过 TapTap SDK 收集 OAID 用于"广告效果分析"，且未提供关闭选项） |
| 是否未提供关闭定向推送/广告功能的选项 | **是**（未提供关闭 OAID 收集或广告追踪的选项） |

**综合评估**：应用本身无广告和定向推送功能，合规风险主要来源于第三方 SDK（TapTap SDK）的 OAID 收集行为。建议优先完成"建议一"和"建议二"，以符合工信部专项整治要求。
