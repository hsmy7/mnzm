# 宗门任务系统设计方案

## 一、设计目标

1. 四种难度全覆盖：简单、普通、困难、禁忌
2. 奖励差异化：根据难度递增，材料稀有度与难度匹配
3. 奖励丰厚程度与难度正相关
4. 弟子准入规则严格化：按难度限制弟子类型和境界
5. 执行弟子数量统一为6人
6. 任务类型多样化：无战斗、必战斗、概率突发战斗
7. 战斗任务成败由战斗胜负决定，非战斗任务必定成功

## 三、任务类型定义

| 类型 | 说明 | 成功条件 |
|------|------|----------|
| NO_COMBAT | 无战斗任务 | 时间到必定成功，发放完整奖励 |

## 七、弟子校验规则

```kotlin
validateDisciplesForMission(mission, disciples):
  1. 人数必须为6人
  2. 所有弟子 status == IDLE
  3. 简单: discipleType == "outer" (外门)，无境界限制
  4. 普通: discipleType in {"outer", "inner"} 且 realm <= 7 (金丹及以上)
  5. 困难: discipleType == "inner" (内门) 且 realm <= 5 (化神及以上)
  6. 禁忌: discipleType == "inner" (内门) 且 realm <= 3 (合体及以上)
```

## 八、任务完成逻辑

```kotlin
processMissionCompletion(activeMission, disciples):
  IF 任务类型 == NO_COMBAT:
    必定成功，发放完整奖励

  IF 任务类型 == COMBAT_REQUIRED:
    调用 BattleSystem 进行战斗（妖兽/人型敌人）
    IF 战斗胜利(TEAM):
      发放完整奖励
    ELSE:
      无奖励，弟子可能受伤/死亡

  IF 任务类型 == COMBAT_RANDOM:
    随机判定是否触发战斗(triggerChance)
    IF 未触发:
      发放基础奖励(灵石30% + 少量材料)
    ELSE:
      调用 BattleSystem 进行战斗
      IF 战斗胜利:
        发放完整奖励
      ELSE:
        无奖励，弟子可能受伤/死亡
```
